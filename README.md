# Walk Daily

A free AI-powered Christian Progressive Web App for daily Bible reading, prayer, journaling, and spiritual growth.

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/your-org/walk-daily)

## Features

- **Daily Bible Verse** — Curated verse of the day with 365-day reading plan
- **Bible Reader** — Full Bible with book navigation, search, highlights, and bookmarks
- **AI Faith Chat** — Chat with an AI assistant about Scripture, theology, and prayer
- **Prayer Wall** — Share and support prayer requests from the community
- **Faith Journal** — Daily reflections with mood tracking and verse tagging
- **Verse Memory** — Spaced repetition system for Scripture memorization
- **Dark Mode** — System-aware theme with smooth transitions
- **PWA** — Installable on mobile and desktop, works offline

## Tech Stack

| Layer        | Technology                              |
| ------------ | --------------------------------------- |
| Framework    | Next.js 14 (App Router)                 |

| Hosting      | Vercel                                  |

## Quick Start

### Prerequisites

- Node.js 18+
- npm or yarn
- A Supabase project (free tier works)
- An OpenRouter account (free tier works)

### 1. Clone and Install

```bash
git clone https://github.com/your-org/walk-daily.git
cd walk-daily
npm install
```

### 2. Environment Variables

```bash
cp .env.example .env.local
```

Edit `.env.local` with your values:

| Variable                        | Where to get it                                   |
| ------------------------------- | ------------------------------------------------- |
| `NEXT_PUBLIC_SUPABASE_URL`      | Supabase project settings > API                   |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase project settings > API                   |
| `SUPABASE_SERVICE_ROLE_KEY`     | Supabase project settings > API (keep secret!)    |
| `NEXT_PUBLIC_OPENROUTER_API_KEY` | [openrouter.ai/keys](https://openrouter.ai/keys) |
| `BIBLE_API_KEY`                 | [scripture.api.bible](https://scripture.api.bible) |

### 3. Set Up the Database

1. Go to your Supabase project > SQL Editor
2. Run the contents of `supabase/schema.sql`
3. Run the contents of `supabase/seed.sql`

### 4. Run Locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

### 5. Build for Production

```bash
npm run build
npm start
```

## Database Schema

All tables use Row Level Security (RLS):

| Table               | Purpose                          |
| ------------------- | -------------------------------- |
| `profiles`          | User profiles (extends auth.users) |
| `waitlist`          | Pre-launch email capture         |
| `verse_marks`       | Bookmarks & highlights           |
| `reading_progress`  | Reading plan tracking            |
| `chat_messages`     | AI chat history                  |
| `journal_entries`   | Faith journal                    |
| `prayer_requests`   | Prayer wall                      |
| `prayer_comments`   | Comments on prayer requests      |
| `verse_memory`      | Spaced repetition memorization   |
| `reading_plans`     | Reference data                   |
| `daily_verses`      | 365 curated daily verses         |

## Deployment

### Vercel (Recommended)

1. Push to GitHub
2. Import the repo in [Vercel](https://vercel.com/new)
3. Add all environment variables from `.env.example`
4. Deploy

### Environment Variables in Vercel

Set these in your Vercel project settings:

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- `NEXT_PUBLIC_OPENROUTER_API_KEY`
- `BIBLE_API_KEY`
- `BIBLE_API_URL`
- `ADMIN_EMAIL`
